
# Ex12 Проект TimedDoor (дверь с таймером)

## Формулировка задачи

Необходимо разработать реализации для системы классов, моделирующих работу двери холодильника с таймером. Дверь должна находиться в закрытом состоянии. Если она открыта, то через заданное время объект двери должен генерировать исключение **close the door!**.

## Классы проекта

Предлагается три основных класса:

- **TimedDoor** - дверь с адаптером.
- **DoorTimeAdapter** - адаптер двери.
- **Timer** - устройство для отсчета времени.

## Описание задачи

В этой задаче предполагается использовать ОДИН поток управления. В начале создается дверь и  адаптер для электронного управления. При открытии двери, создается экземпляр таймера и ему передается адрес адаптера. Через заданный промежуток времени, таймер вызывает метод **Timeout** адаптера, который, в свою очередь вызывает **DoorTimeOut** двери. Последний генерирует (выбрасывает) исключение.

У двери также должен быть реализован метод **throwState**, который выбрасывает исключения **the door is closed!** и **the door is opened!** в зависимости от текущего состояния.

## Структура проекта

- **include/TimedDoor.h** - заголовочный файл.
- **src/TimedDoor.cpp** - файл с реализацией методов классов.
- **src/main.cpp** - домонстрационная программы для работы с классами.

## Алгоритм выполнения работы

Для выполнения работы необходимо:

1. Выполнить *fork* репозитария в свой аккаунт.
1. Выполнить клонирование репозитария из своего аккаунта к себе на локальную машину (`git clone`).
1. Создать ветку **git** с индивидуальным номером (`git branch имя_ветки`).
1. Сделать ветку активной (`git checkout имя`).
1. Необходимо разместить как исходные файлы с решениями задач, поместив **cpp** файлы в **src**, а заголовочные - в **include**.
1. Добавить файлы в хранилище (`git add`).
1. Выполнить фиксацию изменений (`git commit -m "комментарий"`).
1. Отправить содержимое ветки в свой удаленный репозитарий (`git push origin имя_ветки`).
1. Создать пул-запрос в репозитарий группы и ждать результата от **GitHub Actions**.
